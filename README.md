## 3. Краткие теоретические сведения
Операция GEMM (General Matrix Multiply) является базовой операцией линейной алгебры и описывается формулой:
$$C = \alpha \cdot A \cdot B + \beta \cdot C$$
Где $A, B, C$ — матрицы, а $\alpha, \beta$ — скаляры. В данной работе $\alpha=1.0, \beta=0.0$.
Для ускорения вычислений используется библиотека OpenBLAS, которая оптимизирована под конкретные архитектуры процессоров с использованием SIMD-инструкций.

### 4.2. Реализация алгоритма
Для выполнения задания была разработана шаблонная функция `my_gemm`, поддерживающая работу с типами `float` (одинарная точность) и `double` (двойная точность). 

**Ключевые особенности реализации:**
* **Многопоточность:** Распараллеливание вычислений реализовано с помощью технологии **OpenMP**. Директива `#pragma omp parallel for` применена к внешнему циклу по строкам матрицы, что обеспечивает равномерное распределение нагрузки между ядрами процессора.
* **Параметры компиляции:** Сборка программного кода осуществлялась с использованием флага оптимизации `-O3`, что позволило компилятору `g++` применить автоматическую векторизацию циклов.
* **Библиотека сравнения:** Использовалась статическая сборка библиотеки **OpenBLAS** (параметры сборки: `NO_SHARED=1 ONLY_CBLAS=1 NOFORTRAN=1`), линковка которой осуществлялась через `Makefile`.

### 4.3. Описание тестового стенда
* **Размер матриц:** $2000 \times 2000$ элементов.
* **Методика замера:** Для каждого количества потоков ($1, 2, 4, 8, 16$) проводилось по 10 итераций вычислений.
* **Обработка данных:** Итоговое значение времени рассчитывалось как **среднее геометрическое** для минимизации влияния случайных системных задержек на результат.
